本研究根據使用案例文件規劃系統架構，並以系統架構圖定義系統模組及其關聯性。系統架構圖如圖 23 所示，採用 Web Service 架構，考量因素如下：一、執行效能：由於用戶端手機效能不一，建置 Web Service 統一處理來自 App 的服務請求，呼叫 Google Places API 取得搜尋結果後，計算方位角和距離，再將處理結果傳送至用戶端。二、易於除錯與維護：Web Service 比 Unity3D 更易測試，因其輸入和輸出內容皆有介面規格書明確定義。三、未來擴充性：Web Service 不僅能處理 App 的服務請求，也能處理來自網頁的請求，未來可透過網頁呼叫此 Web Service，同樣取得 Google Places API 搜尋結果。 (圖 23 系統架構圖 資料來源：本研究)
