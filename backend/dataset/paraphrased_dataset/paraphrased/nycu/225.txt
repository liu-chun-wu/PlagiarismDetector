以 D-Link COVR-2202 這款 Mesh 網路家用閘道器為例，說明第二類產品的自動化測試實作。市售版本為兩台一組，購買後即可建置 Mesh 網路。按照使用者導向的新產品導入自動化測試流程，分為五個步驟：

1. **新增產品環境參數：**在 twina_config.json 檔案中設定 COVR-2202 的環境參數，包含兩台設備的 console 連線數據。

2. **新增產品參數：**在 device/ 資料夾下建立 covr2202.py 檔案，定義產品參數，並加入針對單台設備 (CAP 或 RE) 的操作。

3. **新增或修改測試步驟 Lib：**將 GUI 操作轉換為自動化程式，建立 CovrGuiLib.py 檔案。

4. **撰寫測試項目：**以 Pre Test 測試項目為例，編寫 Covr2202_TestCase.py 檔案，實現自動化測試。

5. **新增或修改測試項目分類類別：**將 COVR-2202 Pre Test 測試項目添加到 testsuites.cfg 檔案。

完成以上步驟後執行自動化測試即可獲得測試報告。COVR-2202  Pre Test 自動化測試耗時 0.55 天，手動測試和報告整理約 1.5 人天，總測試時間為 1.5 人天 (自動與手動並行)。相較於原本 6 人天 (2 天 * 3 位工程師) 的手動測試，節省了 75% 的人力。


D-Link COVR-2202 的自動化測試開發歷時約 11 個月，包含其他功能的自動化開發以及 D-Link DWR-972 Pre Test 測試項目的自動化開發。COVR-2202 Pre Test 自動化測試開發時間約為 5 個月。
