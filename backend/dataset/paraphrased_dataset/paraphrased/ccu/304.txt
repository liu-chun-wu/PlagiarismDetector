本研究提出三個模組以實現通用系統開發功能，並將ORM概念從原生資料庫操作轉換為一般功能維護和自動化稽核。以下列出各模組可防範的安全漏洞及運作流程。

**資料庫系統模組**

可防範：資料庫注入攻擊、機敏資料洩漏、SQL撰寫錯誤。

運作流程：新增、修改、刪除流程皆為：呼叫方法 -> 檢查參數 -> 組合PDO SQL語法 -> PDO傳輸並接收DB回傳訊息 -> 回傳DB訊息。呼叫預存程序流程亦同，僅SQL語法改為呼叫程序語法。


**系統檔案維護模組**

可防範：惡意檔案上傳阻斷、系統架構探索、系統檔案控管、引入程式控管、系統環境組態控制。

運作流程：
1. 檔案上傳：呼叫方法 -> 檢查檔案副檔名 -> 檢查檔案大小 -> 上傳檔案 -> 回傳執行結果。
2. 檔案刪除：呼叫方法 -> 檢查檔案是否存在 -> 刪除檔案 -> 回傳執行結果。
3. 讀取檔案：呼叫方法 -> 檢查檔案是否存在 -> 開啟檔案並擷取內容 -> 回傳資訊。
4. 檔案編修：呼叫方法 -> 檢查是否存在 -> 進行檔案編修 -> 回傳執行結果。


**參數稽核模組**

可防範：異常存取、XSS攻擊。

運作流程：
1. 參數接收：呼叫方法 -> 判斷來源是否合法 -> 判斷參數傳輸方法 -> 過濾參數值與標準化 -> 接收參數並封裝。
2. 參數使用：呼叫方法 -> 取得通過稽核流程並封裝之變數。
