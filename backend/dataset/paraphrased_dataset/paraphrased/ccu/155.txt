節點信譽越高，代表其過去轉發訊息越多。BS node 計算節點信譽，並依靠鄰居節點提供數據計算RV值，以防止節點偽造訊息。多個數據源確保RV值的準確性。

信譽系統特點：

1. 使用watchdog偵測周圍節點傳輸活動。
2. 維護Near Node List(NNL)記錄周圍節點訊息，包括節點編號、RV值和CV值，並定期更新。
3. 使用watchdog記錄周圍節點的Requested (R) counter和Forwarded (F) counter。
4. RV值計算公式： RV(𝑗) = (∑ (𝑙𝑜𝑔2𝐹𝑖(𝑗) / 𝑙𝑜𝑔2𝑅𝑖(𝑗)) × 𝑅𝑉𝑡𝑎𝑏𝑙𝑒(𝑖)) / 𝑛 ,  其中𝑅𝑉𝑡𝑎𝑏𝑙𝑒(𝑖) ≥ 0.8, 𝑖≠𝑗  (𝐹𝑖(𝑗): 節點i記錄的節點j的Forwarded數量；𝑅𝑖(𝑗): 節點i記錄的節點j的Requested數量；𝑅𝑉𝑡𝑎𝑏𝑙𝑒(𝑖): 節點i的RV值)。
5.  𝑅𝑉𝑓𝑖𝑛𝑎𝑙(𝑁) = 𝛼𝑅𝑉𝑡𝑎𝑏𝑙𝑒(𝑁) + (1 −𝛼)𝑅𝑉(𝑁) ，用於修正封包遺失或偵測不到導致的RV值不準確。
6.  選擇轉發節點的標準：𝑅𝑉(𝑖) + (1 − 𝑑𝑖𝑠𝑡(𝑖) / 𝑑𝑖𝑠𝑡𝑟𝑒𝑔)  (𝑑𝑖𝑠𝑡(𝑖): 節點i與目標節點的距離；𝑑𝑖𝑠𝑡𝑟𝑒𝑔: 請求節點與目標節點的距離)。只參考表現良好節點提供的數據，避免整體RV值下降。

信譽計算步驟：

1. BS蒐集節點N鄰居的watchdog數據。
2. 根據鄰居記錄的節點N傳輸歷史和鄰居可信度計算local value。
3. 所有鄰居的local value相加除以個數得到節點N的RV值。
4. 定期對所有節點執行上述步驟。

信譽系統運作例子：

1. 節點N檢查NNL中節點A的RV值。
2. 若無節點A訊息，則向BS請求。
3. BS蒐集節點A周圍節點的傳輸歷史。
4. BS計算節點A的RV值。
5. BS將節點A的RV值傳給節點N，並記錄在NNL中。
6. 節點N根據節點A的RV值決定是否轉發訊息。NNL數據定期更新。
