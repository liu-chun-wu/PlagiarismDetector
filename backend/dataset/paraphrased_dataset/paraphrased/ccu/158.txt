網路初始化設定範圍為100*100，劃分為10*10的區域，共100個。區域編號[row, column]，row和column值介於0到9。例如[1, 9]代表x軸範圍10到19，y軸範圍90到99，位於第二列第十行。

每個區域中心點設有一個固定、永在線且可靠的BS節點。例如[1, 9]的BS節點座標為[15,95]。BS節點偵測範圍為15，並儲存區域內節點名單。

一般節點分散於網路中，初始RV為0.8，初始CV為50，CV上限為1000。節點依行為模式分為合作、自私和狡猾節點。節點移動速度：40%固定不動，30%速度為1，30%速度為2。

檔案以MinHash技術的LSH方法分群。以文件S1={I, an}；S2={eat}；S3={He, an, apple}；S4={I, eat, an}為例：

1. 將文件轉換為0和1的矩陣。

2. 建立轉換函數 ℎ(𝑥) = (𝑎𝑥+ 𝑏) mod 𝑝，p為最接近文件數量的質數，0 < 𝑎 < 𝑝，0 <= b <= 𝑝。轉換函數數量依實驗決定。例如ℎ1(𝑥) = (𝑥+ 1) mod 5，ℎ2(x) = (3𝑥+ 1) mod 5。

3. 將0到4代入ℎ1(𝑥)和ℎ2(x)。

4. 建立signature matrix，比較文件元素與h(x)值，以較小值更新signature matrix。

5. 將n個轉換函數分為b個band，每個band有r行 (n = b x r)。將同一band中數值相同的文件放入同一bucket。若兩個文件被分配至同一bucket次數超過設定值，則視為相似文件。例如S1和S4在兩個band中都被分配至同一bucket，因此S1和S4相似。

文件分群後，各區域儲存不同群的文件Metadata。源節點將Metadata傳送至指定區域。
