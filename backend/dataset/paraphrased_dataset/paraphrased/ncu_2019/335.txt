本章節根據4.2的分析，聚焦探討如何透過無線網路，在伺服器BMC系統和Linux架構的智能家電系統之間傳遞資訊。軟體設計重點及步驟如下：

1. **Sensor Number/Sensor ID 重新規劃**:  重新檢視並編排Sensor Number/Sensor ID的分類和依據，使其分配更清晰且具規劃性。本研究實驗設計使用7Fh至83h的Sensor Number/Sensor ID，預留的連續區塊也提供使用者和開發者彈性運用。

2. **利用可自行運用的Sensor Number/Sensor ID 建立管理項目**: 使用預留的Sensor Number/Sensor ID 建立管理項目，讓使用者和開發者在BMC系統運行後，能自行調整管理項目的細節並依需求選擇。本研究使用MegaRAC® SP-X BMC Firmware預先建立五個簡單的管理項目 (詳見圖4-2和表4-3)。

3. **設計與開發智慧居家或外部系統所用之指令**: 智能家電和外部系統將透過特別設計的指令與BMC系統傳遞訊息，達成資訊共享。指令設計內容詳見表4-4。

4. **BMC系統內的資訊整合和處置**: BMC系統接收指令後，會將網路延遲容許值、裝置或感知器資料，以及對應的Sensor Number/Sensor ID導入管理項目。關鍵在於網路延遲容許值的使用，以及感知器或裝置數值的取代和獲取。BMC系統以每秒為一個監控循環(Monitor Cycle)，每次循環後，以指令取得的數值取代現有數值。網路延遲容許值則用於判斷數據延遲是否在容許範圍內。若在容許範圍內，則管理項目數值維持上一次有效數值，避免BMC系統因未及時獲取新數據而誤判。本研究透過增加程式變數暫存有效數據和記錄未獲取數據的循環次數，並與網路延遲容許值比較，達成此目的 (程式流程示意圖見圖4-3)。
