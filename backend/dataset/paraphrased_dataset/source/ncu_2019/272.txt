從現況中可以看出，在 L 銀行的 EUC 規模是非常龐大，並且是以 Excel 為主，而在 L 銀行的 EUC 優點可以分為以下四點： 1. 解決部分的業務問題： 對於行員而言，當其他的選擇是相當沒效率的土法煉鋼方式、抱持僥倖心態 的忽略問題，且總行無法提供良好協助時，有效、正確的 EUC 是相對較好 的解決方式，而由於行員是業務與系統第一線的使用者，也是對業務運作最 了解的領域專家，所以可以客製化的 EUC ，許多時候在良好正確的設計與 製作下，成品的效果與功能甚至是非常優良的解決方式。 2. 透過實驗來創新不同的解決方式： EUC 是行員對想法進行實驗並製成成品，在實驗的過程中行員才能對於想 法有更深的了解與產生新的想法，絕大多數創新想法也是在實驗中所產生 的，而若是未經實驗的想法很容易是不完善及缺乏其他方面的考量，所以 EUC 讓行員更加了解問題，以及有更好、創新的解決的方式，並降低行員 在向總行提案時，因對問題與解決方式的了解程度不足或不夠成熟，造成常 見的系統改了又改、改了又再改的困境。 3. 在過程中更加了解業務： 行員在 EUC 的過程中，許多時候會更加了解業務以及相關系統，跟直接操 作與使用系統相比， EUC 會需要製作細節的地方，甚至可能每一個細節與 步驟皆要顧慮到，在如此過程中對於業務與系統的了解會更勝於僅使用系統 的行員，而了解的程度上升，對於行員在業務運作上將更有幫助。 4. 良好的溝通方式： EUC 成品為良好的溝通方式，尤其是與目前 L 銀行常見的文字、語言、圖 片等方式相比，透過成品操作與運用，以及文字、語言、圖片的配合，對於 想法的傳遞才能夠有效的表達與理解，並能降低因為溝通不良而讓創新想法 石沉大海的問題。 而目前在 L 銀行的 EUC 運用問題上，主要有以下兩點： 1. 行員 EUC 能力有待提升： 行員的資訊能力有限，主要以 Excel 為主，對於程式語言皆不熟悉， EUC 時常常會因此受到限制，即便有好的想法卻無足夠能力去發揮運用，僅能運 用 Excel 及人工方式來解決問題，所以需要提升行員的能力，讓行員能更有 效率地去發揮 EUC 。 2. EUC 無正式交流管道： 因為 EUC 時常是個別行員的行為，許多成品並無交流與分享，即便有交流 也僅限於行員所屬的分行，對於 L 銀行150 家分行的規模，更是造成交流 不易，所以沒有一個正式的 EUC 交流管道，會造成許多問題，如容易對於 相同問題，卻重覆耗費不同行員的時間精力去建置 EUC 來解決，同樣的事 情卻不斷重覆人力，這是非常沒有效率，且對於 L 銀行是非常大的潛在成 本； EUC 交流少會讓行員間降低共享的成果，因為每個行員的領域知識、 EUC 能力、解決問題能力、程度、時間、精力等皆不相同，沒有交流讓遇 到相同問題但卻無法 EUC 解決的行員，無法共享 EUC 的成果來增進業務 運作效率，這對於重團隊合作的分行業務效率將大打折扣；而交流少也造成 行員的 EUC 是個別在建置與使用，僅是個別行員的想法與角度，較少其他 行員的觀點、想法與意見，如此使 EUC 的成果有限，明明在團體工作卻無 法運用團體優勢，遇到問題時也容易沒有人幫忙，非常的可惜。 所以如何改善這兩個問題，來發揮行員 EUC 的優點，讓業務運作更加順利， 是一大課題，並整理成如表3-1 與表3-2 所示。 表3-1  EUC 優點 項目 內容 1. 解決部分業務問 題 有效的 EUC 可以解決許多行員遇到的問題，讓業務能順利運 作。 2. 透過實驗來創新 不同的解決方式 絕大多數的創新與改變皆是從實驗中產生，透過 EUC 能讓行 員進行實驗，並更了解業務與系統。 3. 在過程中更加了 解業務 行員在 EUC 的過程中，需要以不同的角度與方式來重新檢視 與思考業務運作，所以在過程中會更加了解業務。 4. 良好的溝通方式 比起僅用語言、文字來溝通，用 EUC 成品配合語言與文字的資 訊傳遞，才能良好的表達與交流。 資料來源 : 本研究整理 表3-2  EUC 運用問題 項目 內容 1. 行員 EUC 能力 有待提升 行員因普遍僅會 Excel ，且並無相關訓練，使 EUC 常常遇到 限制，安排適當訓練有助於提升行員 EUC 運用。 2.
