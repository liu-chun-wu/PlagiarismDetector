Fabric 全名為Hyperledger Fabric，原先為IBM 發起，並聯同Intel、Cisco 等 公司合作的企業級區塊鏈框架，旨在開發採用模組化架構的區塊鏈應用或解決方 案，整個項目在2015 年轉移給 Linux 基金會托管，成為開源的專案。 在Fabric 中，有兩種類型的節點，分別是同伴節點(Peer)以及排序節點(Orderer)。 • 同伴節點(Peer)：可部署鏈碼(chaincode)，對帳本進行讀寫操作，並同時具 有背書節點(Endorser)與提交節點(Committer)的功能，將在以下交易流程提 及。 • 排序節點(Orderer)：對交易進行排序，將交易批次組合成區塊，傳送給 同 伴節點。 在Fabric 中，會以通道(Channel)區分不同的節點與使用者，通道(Channel)以 群組為概念，不同群組提供不同的商業服務，不同組織可以節點為單位，選擇參 與不同的業務。整個Fabric 的交易流程如下: • Enroll 客戶端(Client)：使用用戶(User)帳號向節點(Peer)註冊(Enroll)取得用戶憑證 (Keys)。 • Proposal 客戶端(Client) ：提出交易提案(Proposal) ，傳送背書節點(Endorser) 建立交易提案(Propose)。 ➢ 背書節點(Endorser)：背書節點在收到交易提案後，會驗證數位簽章， 確認用戶權限，再來執行交易，產生交易結果，最後回傳資訊給客戶 端。 • Submit Transaction 客戶端：透過同伴節點 (Peer)將交易廣播到排序節點(Orderer)，背書節點 (Endorser)將訊息廣播發送給排序節點(Orderers)。 • Relay 背書節點 (Endorser)將訊息廣播發送給排序節點 (Orderers)。 • Batch 排序節點 (Orderers)執行共識驗證，交易排序，並產生區塊。 • 提交節點(Committer)：將資料寫入區塊鏈，驗證交易的狀態是否相符。 圖18  Hyperledger Fabric 架構 資料來源 : https://www.hyperledger.org/
