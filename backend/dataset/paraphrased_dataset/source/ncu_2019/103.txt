根據前兩節的敍述，本研究認為機器學習具有解決重複輸入報案資料、筆錄自動分 類等問題的能力，並有提供筆錄問答句推薦與移送書推薦潛力，所以本節提出一個應用 機器學習技術協助警察調查詐欺案件系統（A Fraud Investigation Police Assistant System by Machine Learning Approach,簡稱FIPAM）來支持本研究看法。該系統前端 為「筆錄制作介面」與「分析視窗介面」，分別建置於即將上線之「刑事案件管理系統」 與目前的「165 反詐騙系統」；中端有自然語言處理模組、分析數據引擎、資料搜集器 模組、筆錄推薦模組、移送書推薦模組等；後端有「165 反詐騙系統」資料庫、筆錄資 料庫、犯罪資料庫、移送書資料庫、使用者行為資料庫等。利用FIPAM 可應用於筆錄制 作推薦問句、結構化資料分析、關鍵字分析、關聯式分析、智慧推薦相似案件 、實時分析詐欺犯罪等功能，FIPAM 之架構如下圖3-8。 圖 3-8 FIPAM 架構 【圖片來源：本研究整理】 第55 頁 FIPAM 在運作前，為了使「165 反詐騙系統」資料庫中的筆錄作正確且符合需求的 分類，須先訓練模型，如圖3-8，透過標記方式幫LDA 模型建立主題標記。首先建立筆 錄訓練集，以「165 反詐騙系統」資料庫最近10 年報案紀錄留存超過30 萬件筆錄作為 訓練資料來源，進行中文斷詞、刪除無意義的停止詞、建立新詞彙等自然語言處理，以 完成非結構化數據預處理。第二步，建立低層次語義的資料庫，利用前述30 萬件筆錄 作為訓練資料作為Word2Vec 訓練詞崁入（Word Embeddings），建立符合調查筆錄與法 律用語之語料庫，利用LSTM 演算法找出具有時序特徵的筆錄上下問答句詞彙意圖。第 三步，建立高層次的筆錄—主題—關鍵詞標籤，利用LDA 模型找出潛在主題，再由各主 題的重要詞彙作人工標記、修正詞彙分類，再為各主題建立中文分類名稱。第四步，可 依需求選擇詐欺方法、詐欺管道、支付方法等不同面向作為分類的主題。 圖 3-9 訓練LDA 模型示意圖 【圖片來源：本研究整理】 以上，第一步及第二步係為較低層次的語義建立準確之關鍵，尤其將法律用語、詐 欺術語、行話與被害人案情描述詞彙作刑案偵查特徵詞彙的向量，建立詞彙標籤的語料 庫。將來LDA 模型就能有效判斷筆錄問答句詞彙所描述的主題分布，進而作協同過濾推 薦。以下介紹FIPAM 架構各部，並舉例說明如何運作。 第56 頁
