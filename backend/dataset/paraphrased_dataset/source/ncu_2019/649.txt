LabVIEW 是一款專為測試、量測與控制應用所設計的系統工程軟體，它也是能圖 型化編輯流程，程式設計師通過繪製導線連接不同功能的節點，設計完圖形化的程式框 圖結構來決定程式如何執行。 運用軟體LabVIEW 解決工程應用的問題時，以圖2.3.1.a LabVIEW 操作與開發架 構圖，DSL developer 藉由LabVIEW 開發DSL 時，LabVIEW 的擴充可以藉由formula node、SubVI，或是外掛模組或工具組，進行開發DSL。formula node 是基於script 撰寫 程式碼建構的node，可以使用C 語法結構在程式塊上編寫，達到執行的數學運算，如 圖2.3.2.a。可以針對所需之運算自定義formula node 內容，在formula node 中設定輸入 參數、返還值、執行內容的方式進行擴充[18]；SubVI 是基於子程式上加以建構的，子程 式被稱為虛擬儀器（Virtual Instrument，簡稱VI），如圖2.3.2.b，NI LabVIEW 軟體的強 大之處在於VI 的分層特性。創建VI 後，可以在另一個VI 的程式塊中使用它[19]。層次 結構中的層數沒有限制。使用模塊化編輯程式可幫助快速管理更改並使用，也就是說可 以用多個子程式VI 建構出一段程式後，將其合併成一個SubVI，得到自定義的新VI， 以達到擴充新功能。但如此能創建擴充出一系列新的SubVI 達到建構該領域所需的DSL。 DSL developer 利用LabVIEW 開發DSL，能以formula node 或是新建VI 或SubVI， 建構該領域所需DSL。formula node 提供空白的script 提供撰寫程式碼，script 中需設定 輸出入變數、變數型態、執行動作、運算內容……等; 新建VI 也需要設定VI 屬性、設 置前面板Icon 樣式、或是在程序框圖藉由現有VI 拼組合併成SubVI，或創建新VI 時， 創建的功能方塊VI 也需要設定外型、輸出入端口數量、各端口變數資料型態、執行內 容……等，這些設置能讓DSL developer 開發時更加嚴謹，相對的開發過程也會需要花 費不少開發時間。 LabVIEW 建構的DSL 多應用於工程模擬與量測，元件多以簡單的電子控制元件或 邏輯運算元件，但代換領域時，可能生物、經濟、化學等，他們可能無法直覺了解電子 元件圖式所代表的運算，所以實際上非工程領域上的應用會比較不直觀。 圖2.3.2.a  formula node 範例圖[20] 圖2.3.2.b  SubVI 範例圖[19] 三、研究提案 以許多領域常使用流程圖，所以流程圖為構想，建構流程圖式的小型DSL 概念[21]， 提供End-User 簡單的編輯流程圖，可以簡易的編排你的步驟流程、設定條件判斷、循環 次數等。就能夠建構、操作目標程式，且提供自定義功能塊功能予以DSL developer 建 構該領域的DSL，通過簡單的變數和數學式設置，建立該領域常使用的函式或執行動作 行為，甚至因為流程圖在各領域的使用的普及性，方便讓多方領域也能以此概念建構其 領域的DSL。
