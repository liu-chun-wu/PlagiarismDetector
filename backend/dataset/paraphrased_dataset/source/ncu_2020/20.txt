因透過Wireshark 的解析發現不同廠牌攝影機於傳送影像串流時也會順帶夾帶非影 像相關的Non-VCL-NALU 如SPS，PPS，SEI 於其中，其所對應到的RTP Timestamp 數值 因為會和後一個相鄰的video frame 所對應到的RTP Timestamp 相同，導致於封裝成MP4 後於video track 對應到stts box 中產生delta time 為0 的項目資訊。stts box 內紀錄每個video frame 需要於畫面上停留多久的關鍵資訊，而該資訊會與media server 在收容前端攝影機 串流時攝影機於RTP header 中所攜帶的timestamp 欄位資訊有關，本研究驗證海康、晶 睿、安迅士、奇偶、大華、利凌廠牌攝影機所產生的視訊流，根據國外研究報告（Tschudi, 2020）前述廠牌涵蓋超過70%的市佔率，透過本研究的修正流程可將其MP4 錄影檔在不 需重新編解碼的情況下修正成可直接透過網頁瀏覽器中原生HTML5 播放器直接播放以 達到於單一平台上收容與播放不同廠牌攝影機之歷史串流目的。
