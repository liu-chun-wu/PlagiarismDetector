在實驗環境中，我們在Server & Client 上安裝SFTP（採用TCP）、SAMBA（採用TCP） 與Tsunami（採用UDP），來比較三種傳輸軟體的效能差異。 (1) SFTP 安裝 執行以下指令安裝openssh-server $ sudo apt-get install openssh-server $ dpkg –get-selections |grep ssh (2) SAMBA 安裝 執行以下指令安裝SAMBA $ sudo apt-get install samba 設定SAMBA $ vi /etc/samba/smb.conf 於檔案尾端加入 [share] #分享名稱 path = /home/richou/share #分享資料夾路徑 writable = yes guest account = “username” force user = “username” public = yes 重新啟動SAMBA Server 讓設定生效。 $ sudo /etc/init.d/smbd restart 於Client 上掛載Server 分享的資料夾 $ sudo apt install cifs-utils  #安裝cifs-utils $ sudo mkdir /mnt/share  #建立mount point $ sudo mount -t cifs -o username=“username”, password=“password” //192.168.190.130/share /mnt/share  #掛載Server 分享的資料夾 (3) Tsunami 安裝 Tsunami 只提供原始碼必須以自行編譯的方式安裝，首先要安裝一些編譯套件， 如gcc, git, automake, autoconf 與 make. $ sudo apt install gcc git automake autoconf make #安裝編譯套件 從git 上下載Tsunami 原始程式碼 $ git clone https://git.code.sf.net/p/tsunami-udp/code_git tsunami-udp-code_git 編譯Tsunami $ cd tsunami-udp-code_git/tsunami-udp/ $ ./recompile.sh 安裝Tsunami $ ./configure --prefix=/opt/tsunami #指定安裝路徑 $ sudo make install #進行安裝 啟動Tsunami server $ cd /home/richou/share  #先進入檔案分 享的路徑 $ /opt/tsunami/bin/tsunamid --hbtimeout=3600 * #啟動服務並設定timeout 值為3600 秒，避免封包遺失過於嚴重，超過預設的15 秒而斷線。 啟動Tsunami 客戶端 $ /opt/tsunami/bin/tsunami tsunami> set rate 100M  #設定頻寬目標為100Mbps tsunami> connect server  #連接Server
