一般網路攝影機於RTSP DESCRIBE 回應內返回後續使用RTP 協定之連接埠資訊， 以供訪問端知道後續該使用何種連線方式取得以使用H.264/AVC 壓縮技術產生之影像 資料。攝影機可以決定串流傳送是走在現有RTSP 上的既有TCP 連線，或是另外指定新 的UDP 連線方式來供訪問端存取串流。訪問端接收到串流資料後如要封裝成為歷史影 像檔案，則須依照ISO/IEC-14496-12 ISO Base Media File Format 標準規範進行Meta Data 資料新增後才可以MPEG-4 檔形式儲存，串流於傳輸時其資料會相依於RTP Header 內之 資訊，以及RTP Payload 內之Network Abstraction Layer Units（NALU）內所紀錄之NALU Type 資訊。因H.264/AVC 為影像壓縮技術，其本身並不會帶有時間資訊，換言之要知道 畫面的每一幀（Frame）會停留在畫面上多久，必需仰賴RTP Header 上所標示之時間戳 記，本研究將針對MPEG-4 結構內stss、stsc、stco、stsz 以及stts box 結構資訊對HTML5 播放器之影響進行討論，並歸納出何種封裝方式以目前主流之瀏覽器（Chrome、Firefox） 內之HTML5 播放器才可正常呈現。並在不做重新編解碼的情況下修復原本無法直接於 網頁播放之錄影檔。
