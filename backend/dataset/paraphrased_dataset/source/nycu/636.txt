Camera network 在做檢測和追蹤時，對網路運行有很大的影響。在[5]當中， 提到目標的運動方向正確可以有效減少Camera 之間傳遞訊息的消耗。當目標出 現在某幾個Cameras' FOV，將發現的資訊傳遞給主要的node，讓它決定誰坐定 位。在定位的過程中，為了判斷它的定位方向，會選擇距離FOV 邊緣最遠的方 向，來當目標未來的運動方向，即便當前目標的運動方向是隨機的，選擇距離 FOV 最遠的方向仍能減少不必要的消耗。 為了能夠更加判斷被偵測人物的精準度，[9]提出新的3D 生物識別功能，專 注於重新建構camera network 的框架。使用depth cameras 捕捉目標的depth information。利用3D 背景減法獲得前景目標、透過particle filter 修正圖像變形、 檢測錯誤或半遮擋。若多個目標身高相同，則利用前景目標的depth 形成3D 的 histogram。對於overlapping cameras 會建立一個矩陣，判斷目標出現或離開時。 Non-overlapping camera 利用time-varying，得知目標最後一次得速度和消失的時 間點。如此，Cameras 之間會形成一個PDA(Probabilistic Data Association)的追蹤 問題，所以最後一次出現的camera 得分享feature vector、目標定位、ID、消失時 間等。 由於目標追蹤的難題在於畫面不相交、光線變數。[10]利用spatial and temporal 的資訊來預測下一個camera 的目前狀態，並且使用Q-learning 來做訓 練，但不會預設camera network topology 的任何信息。在目標出現在新的camera 後，會利用一些frames 找到目標位置，並且學習camera topology。限制：1. 時 間的間隔顯示，攝影機交集太少無法學習，epsilon-greedy 對於完全探索沒有太大 幫助。2. 訓練人數越多，就能夠探索更大的狀態空間，但是annotation 成本太大。 3. 需要對labeled set 在線更新進行重新識別，因為一旦識別錯誤的人，會開始追 蹤不同的人。
