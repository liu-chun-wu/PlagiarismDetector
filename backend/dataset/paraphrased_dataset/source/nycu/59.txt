現今Web 前端技術趨近於成熟階段，從視覺體驗到用戶體驗都是 較為優良的，這也是從早年簡單的前端技術(HTML、CSS、JavaScript) 演進到目前複雜的前端框架技術。 市面上的前端框架種類非常繁多，本論文挑出目前市佔最高的前 三名進行比較 ，依序有Angular、React 與Vue，本論文將以2018 年 在GitHub 上的專案數量、工作需求市占率、學習曲線與功能延展性 作為比較。 1. Angular 是一種由Google 支持的、基於TypeScript (由微軟開發的自由和 開源的程式語言)的JavaScript 框架，同時也是一種非常受前端開發者 歡迎的框架，對架構上的要求也比較嚴謹(SPEC INDIA, 2018)。 2. React React 是由Facebook 進行開發與維護網頁架構，其透過構建不同 的UI 組件所提供的JavaScript 函式庫，來協助企業建立其網頁應用 程式。許多的大型企業(例：Uber、Netflix、Twitter、Udemy、Paypal、 Reddit、Tumblr、Walmart)均以React 架構為基礎，來進行其網站之開 發(SPEC INDIA, 2018)。 3. Vue Vue 為採用HTML、CSS，以及JavaScript 來進行開發之架構，其 藉由架構的內部依賴性以及靈活性，來協助使用者用簡易的程式碼來 進行應用程式的開發。其特色是在開發時，套用了JavaScript 的架構， 使網頁應用程式的開發相對簡單(SPEC INDIA, 2018)。 儘管上述三種架構各自在網頁開發市場上佔有一席之地，但因 Angular 的架構提供了完整且嚴謹的開發程序及準則(例：資料庫、路 由方案)，讓後續的維護及更新有了更加標準化且一致的作法，故本研 究後續將以Angular 做為本系統之開發基礎。 而在架構上的學習及維護能力部份，儘管Angular 的學習門檻較 React 及Vue 來得不容易，但其在架構上的規範，使之在更新及維護 上的難度相對較低，其比較表如下圖3-4 所示： 表3-1 前端框架比較表 架構 Angular React Vue 類型 單一框架 要建構UI 函式庫 UI 函式庫 創始者 Google 由Facebook 維護 Google 前員工 學習曲線 陡峭 比Angular 容易一點 平穩的學習曲線 模型 MVC 虛擬文檔對象模型 虛擬文檔對象模型 編寫方式 Typescript JavaScript JavaScript 資料來源：本研究整理 本研究依據系統架構性、易維護性、未來升級性、適合的系統規 模等因素，來評估上述3 個架構，發現因Angular 較適用於架構嚴謹， 且規模較大之系統。因若系統架構嚴謹度不足，或是背後維護及更新 架構的組織規模不足，將導致日後的系統維護及升級上的困難。為此， 本研究將採用Angular 作為系統開發的前端框架，並結合RXJS 數據 傳輸技術與SCSS 壓縮技術，來進行系統前端的開發。
