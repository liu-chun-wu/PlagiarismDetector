隨著網路科技的發達與快速擴展，隨時隨地都有大量資訊於網際 網路上進行傳播。而建構在網際網路上數以千萬計的網站，可說是成 就此一局面不可或缺的力量。舉凡資訊傳播、資料傳遞甚至於商業交 易行為等皆可透過這些網站行為來進行，完全不受時空阻隔的影響。 但在網際網路上傳播的資訊不但種類龐雜且內容繁多，傳統內容固定 的靜態網站設計，在現今資訊提供的時效性、服務內容的多樣性以及 管理維護的方便性已不敷使用(林伯勳，2001)。 由於網頁應用程式的資料倉儲及運算處理皆集中於伺服器端，基 於方便性管理與提高系統執行效率的考量，有必要分散或分離架構之 不同伺服器上以提升效能與穩定度。三層式(3-tier)的分散架構為目前 網頁應用程式最普遍使用的架構。其主要是由三個層級所構成，分別 是：使用者介面層(Presentation tier)、運算邏輯層(Business Logic tier) 及資料服務層(Data Service tier) (林伯勳，2001)。其架構如圖2-3 所 示： (一) 使用者介面層： 在典型的3-tier 架構中，用戶端的使用具備圖形化操作介面(GUI) 之軟體，而這些軟體可以顯示來自伺服器回應之格式化資料，或是透 過特定格式之表單設計供使用者輸入，與伺服器進行資料傳遞之互動 行為。在Web 應用架構中，使用者介面層指的就是諸如IE、Chrome 或是各大移動裝置所以提供的瀏覽器。 (二) 運算邏輯層： 負責運算邏輯的處理程式應於伺服器電腦上執行，其功能是接受 來自用戶端的請求並且決定何種資訊是可以被傳送至用戶端。所以， 運算邏輯層是用來作為使用者與資料庫之間的橋樑，負責資料處理、 商業邏輯處理等運算的工作。微軟利用其作業系統商用軟體極高普及 率的優勢，在網頁應用架構中亦推出各種產品。於運算邏輯層中，微 軟的系列產品包括了IIS(Internet Information Server)網站伺服器，並可 使用ASP(Active Server Pages)設計網頁應用程式，再透過ODBC 或 OLE DB 等機制與各種資料庫媒合。在網路交易及資料傳輸安全認證 等應用，亦有其他相對應的軟體可供其使用。 (三) 資料服務層： 在資料服務層中，包含了儲存大量資料的資料庫，以及用來管理 維護資料的軟體。資料服務層是負責提供資料至運算邏輯層，再傳送 至使用者介面層，所以用戶端無法直接存取資料庫的內容或進行連 線，必須透過運算邏輯層的銜接方可達成，因而提高了系統的安全性， 甚至可以達到高可用性。 三層式架構採用了client/server 運作模式，各個階層可以同時由 不同的團隊並以不同的程式語言或工具進行開發。由於各層級功能的 改變並不會影響到其他層級，所以非常容易在團隊中進行佈署與分 配。在任何一層有必要修改或增加新的功能時，皆不致影響其他層級 的正常作業。 圖2-3 3-tier Web 應用程式架構 瀏覽 網站伺服器及 資料庫伺 使用者介面 運算邏輯 資料服務
