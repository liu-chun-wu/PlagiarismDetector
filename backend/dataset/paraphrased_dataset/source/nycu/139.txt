關於Oracle RAC One Node 切換主機，分為兩種Failover 與Relocate， 此兩種方式皆會造成原有連線中斷，待切換完成後，才能正常使用資料庫。 1. 主機故障切換 (Failover)：模擬方式，直接關閉正在運作資料庫的主機， 觀察RAC 環境另外一台主機是否開始啟動資料庫接管，切換完成由 Windows Client 連線資料庫，檢查運作狀態。 2. 主機維護切換 (Relocate)：通常是需要安裝Patch、硬體維護或既定的開 關機切換排程，才會需要使用此方式。模擬方式，於RAC 環境中使用指 令，移轉資料庫服務到另一台主機上，同樣也是切換完成後由Windows Client 連線資料庫，檢查資料庫狀態，相關操作及結果參考附錄十五。
