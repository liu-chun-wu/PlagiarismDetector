殭屍網路的組成可分為三個部份，如下所述： （一）Botmaster（Bot Herder） 等同司令官的角色，也叫做Bot Herder（牧人），負責下達指令給Botnet 殭屍網 路成員，為駭客本身，稱之為Botmaster，可以透過C&C Server（Command and Control Server）控制其餘殭屍電腦並設法將自己隱藏在幕後，常利用Proxy 的方式偽裝自己的 IP 位置，故很難被找到，藉此規避法律責任。 （二）Bot（Botclient） 不同於一般的病毒及木馬，它可以是一個簡單的網路連線程式，沒有病毒及木馬 的特徵，因此常可以不被防毒軟體偵測到。殭屍程式主要的功能是與C&C Servers 連 絡並接收攻擊命令及程式，殭屍程式本身不是病毒，它的攻擊程式是由C&C Servers 提供。故當被植入殭屍程式時防毒軟體不易偵測到，一旦殭屍程式入侵系統將防毒軟 體關閉或更改，便可從C&C Servers 下載攻擊程式來破壞系統及網路。由於在Bot 潛 伏期時，封包、流量皆與正常時相同，並無明顯差異，因此使用者往往不易察覺已成 為殭屍網路的一份子。 （三）Command and Control Server（C&C Server、C2 Server） 命令與控制伺服器。負責分擔Botmaster 的流量和存放Botmaster 所下的命令，當 Bot 向C&C Server 送出請求和更新時，C&C Server 便擔任傳遞命令的角色，C&C Server 會紀錄Bot 的數量和相關資訊，Botmaster 可從中掌握殭屍網路的狀態。[4] 圖 3 IRC botnet 運作示意圖
