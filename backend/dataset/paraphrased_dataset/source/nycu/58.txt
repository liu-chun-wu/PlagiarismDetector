考量使用者操作該平台之裝置多樣性，且讓使用者使用各種裝置 時，皆方便使用該系統，進而透過此系統來快速地完成機器學習之分 析。 因此，本研究將平台建立在以網頁為基礎之環境中，其優點如下 列所示： 1. 無須安裝任何軟體及特殊之應用程式 2. 大幅降低因裝置環境而造成系統錯誤或失靈的可能性 3. 較高的跨平台通用性，不同的作業系統或電腦設備，皆支援 網頁的瀏覽與操作 4. 藉由功能強大的網頁效果，可提升使用者操作的體驗 儘管以網頁為開發基礎有上述諸多優點，但其亦有少數缺點，如 下列所示： 1. 涉及多種程式語言的開發與設計，須學習的技能較多 2. 因包含由多種程式語言所建構之系統，故須具有多個串接介 面來協助其資料傳遞，亦導致開發過程較為繁瑣 儘管以網頁為基礎之系統架構在開發上較為耗時耗力，但其讓使 用者方便操作的特性，仍深受資料分析系統開發者之青睞。接著，為 使此系統具備有模組化的特性，故本研究將此系統概分為4 個層級， 以提升後續維護、新增、修改功能之效率，分別有Web Layer、Service Layer、Transform Layer，以及 Model Layer。 Web Layer 主要是提供使用者視覺化之設計，僅負責呈現資料或 圖表的輸出結果、相關的顯示參數設定，以及將記錄使用者的設定參 數傳遞至Service Layer。而在進行資料交換時，本研究採用JSON 格 式作為其交換之通用格式，當Service Layer 接收到請求後，除將使用 者參數設定存至資料庫之外，亦會傳遞該資訊至Transform Layer。 最後，當Transform Layer 在接收到JSON 請求後，會將接收到的 資訊轉換成Python 物件，並傳遞至Model Layer，進行資料處理與模 型建立，其軟體及硬體架構如下圖3-1 與圖3-2 所示。 圖3-1 系統軟體架構 圖3-2 系統硬體架構
