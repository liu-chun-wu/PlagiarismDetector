後面的測試會使用到iSCSI 方式與主機介接，所以我們在本章節對常 用的幾種介接方式做簡單的介紹。 (1)  DAS (Direct Attached Storage)直接連接儲存 DAS 介接的方式大多是早期的儲存設備，其元件主要由SCSI 或SAS 介 面的HBA 卡﹙Host Bus Adapter﹚以及磁碟櫃所組成，直接與主機相連， 其效率遠超過當初的ATA 介面。但此磁碟陣列不能與多台主機相連， 僅能透過已連線主機用網路方式分享，彈性與效率已不比現在的NAS 或SAN 的儲存架構。 (2)  NAS (Network Attached Storage)網路附屬儲存 NAS 是儲存設備已內建檔案分享的系統功能，將其附屬的磁碟櫃組成 分享的空間，以Linux 與Unix 的NFS(Network File System)、Windows 的CIFS(Common Internet File System)、iSCSI(Internet Small Computer System Interface，基於TCP/IP 與SCSI 協定下的儲存技術，將原來只 用於本機的SCSI 協定，透過網路的方式掛載為本機磁碟)，成本比SAN 架構要低很多。 (3)  SAN (Storage Attachment Network) 光纖儲存區域網路 SAN 架構是將所有的儲存設備集中於光纖設備SAN Switch 上，類似 於區域網路的概念但與實際的網路是獨立運作的體系，彈性高可集中 管理與主機高速介接。但要與其介接的主機需安裝光纖介面卡，以光 纖纜線為傳輸介質，成本較其它方式都高。可以參考圖11 的儲存介接 方式架構圖，可大致了解其介接方式。 圖11  儲存介接方式架構圖 資料來源：本研究整理 關於安裝主機資源與網路相關規格，可參考下表所示： 表 6   異機還原與第三方磁碟陣列複製測試，需求配置 Hostname 主機名稱 ora1 ora2 freenas2 freenas3 mydns IP 網址 192.168.1.201 192.168.1.202 192.168.1.230 192.168.1.231 192.168.1.88 磁碟陣列 空間 無 無 45G 100G 無 本機作業 系統配置 空間 /boot 1G swap 16G /    43G /boot 1G swap 16G /    43G 安裝自動設 定 安裝自動設 定 /boot 1G swap 4G /    20G Oracle 資料庫版 本 12.2.0.1 12.2.0.1 無 無 無 相關軟體 版本 無 無 無 無 Bind 9.8.2 虛擬主機 平台 VMware VMware VMware VMware VMware Hostname 主機名稱 ora1 ora2 freenas2 freenas3 mydns Workstation 15 Pro Workstation 15 Pro Workstation 15 Pro Workstation 15 Pro Workstation 15 Pro 作業系統  Oracle Linux 6.10 (含更新 2019/06) Oracle Linux 6.10 (含更新 2019/06) FreeNAS- 11.2-U4.1 FreeNAS- 11.2-U4.1 CentOS 6.7 (含更新 2019/06) 記憶體配 置 4G 以上 4G 8G 8G 1G 資料來源：本研究整理
