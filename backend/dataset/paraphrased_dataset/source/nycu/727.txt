依據IEEE 802.11 的定義，有以下兩個基本的網路架構: 1. Ad-Hoc Mode : 由數個Stations(STAs)，例如可攜帶式裝置:筆記型電腦， 同處於一 個小區域，組成Independent Basic Service (IBSS)，彼此之間以點對點 (peer-to-peer)方式進行通訊。在此種模式中，不需要Access Points(APs)的 存在，不同的Stations(STAs)彼此之間既可進行資料的傳輸與交換。由於 此模式的組成元件只由Stations(STAs)所組成，所以其好處是使用者可以 簡單又快速的進行資訊的交換。 圖2-4 80.2.11 Ad-Hoc Mode Architecture 資料來源:NIST, 2013 [11] 2. Infrastructure Mode : 在同一個BSS 單位裡，每個STAs 共用APs 連接至DS 的網路頻寬， STAs 越多，所能享有的網路頻寬就會越小。在企業裡，因區域範圍的廣 大以及使用者人數眾多，所以會建置數個BSS 單位，彼此不同的數個BSS 透過傳輸分配系統DS 來連接，形成Extended Service Set (ESS)的連接方 式，此種模式為一般企業最常建置使用的模式。 圖2-5 802.11 Infrastructure Mode Architecture 資料來源:NIST, 2013 [11]
