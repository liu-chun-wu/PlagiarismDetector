圖 七 分送Metadata 資料來源：本研究整理 Source node 會產生分送metadata 給其他節點，metadata 中包含了檔案的關鍵字和 Source node 的編號和座標(圖七)。 發送Metadata 的流程如下： 1. Source node 產生metadata 並使用LSH(locality sensitive hashing)決定metadata 應 該傳送至哪個區域。 2. Source node 透過聲譽系統(reputation algorithm)和信用系統(credit algorithm)選出 可靠的節點來協助轉發metadata 至該區域的BS node。 3. 當BS node 收到metadata 後，BS node 會先算出該有m個節點會收到此metadata， 且 m = 2 √𝑣𝑖𝑒𝑤 。然後決定forwarding fanouts a 和forwarding level l。a 是每 次轉發該轉發給幾個節點。l 是該轉發幾次，且m = 1 + a + 𝑎2 … … + 𝑎𝑙，藉由位 處於區域中心的BS 分送metadata 可以將訊息平均的散布在區域中，避免有訊息 集中在區域的某些方位中，導致搜尋結果產生錯誤 4. BS node 蒐集所有metadata 位置資料並回傳給source node。 應用於混合式無線網路之信用檢索系統
