移入區域過程如圖五所示： 1. 新節點在移動至新區域後，會選擇最接近的BS 作為自己的BS node，若有兩個 距離相同的BS，則隨機擇一。 2. 在加入此區域時，新節點會通知BS，BS 再將新節點加入列表中。BS 會判斷新 加入的節點是否需要保存metadata。 3. BS 通知範圍內所有節點關於新節點的資訊。 4. BS 將範圍內的節點名單給予新節點，如果新加入的節點需要保存metadata，則 BS 需要再給予新節點source node 的位置。 圖 五 節點移入區域 資料來源：本研究整理 移出區域過程如圖六所示： 1. 節點移出區域前會通知BS，BS 將節點從節點列表中刪除。 應用於混合式無線網路之信用檢索系統 2. BS 通知此區域的所有其他節點，此節點離開。 3. 若此節點持有metadata，會先將metadata 移交給其他節點再離開。 4. 若此節點是requesting node，在會先通知BS 要移動的目的區域，並保存一段期 間，超過時間則BS 刪除此資訊，requesting node 需重新發送請求。 圖 六 節點移出區域 資料來源：本研究整理 應用於混合式無線網路之信用檢索系統
