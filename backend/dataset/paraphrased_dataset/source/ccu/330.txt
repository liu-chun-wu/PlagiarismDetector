資料視覺化是一種將資料以圖像環境幫助資訊接收者理解資料的方法。當今主流 的資料視覺化軟體己經可以幫助人們理解資料可能隱含的模式、趨勢和其他相關性， 大幅超越以往使用Excel 電子試算表中使用的標準圖表和圖形，以更複雜的方式呈現 資料，以及更多的互動功能讓使用者能做及時的鑽入（Drill Down）查詢和分析。 資料視覺化的重要性 當今在BI（Business Intelligence）領域的二家領先廠商為Tableau 和Qlik View，其產品都非常強調視覺化特性，也使得其他供應商在他們的軟體中也紛紛加入 相對應的功能。因此，相較於傳統的統計分析軟體或早期版本的BI 軟體更容易操作。 資料視覺化軟體在大數據分析中也發揮著重要作用。隨著企業在大數據趨勢的早期階 段積累了大量資料，他們需要一種快速有效探查數據的方法。 資料視覺化工具就非 常適合，即使在沒有太多IT 資源的情況下，使用者也能自行操作。另外，當數據科 學家在進行預測分析或機器學習算法時，將成果視覺化以監控並確保模型按預期執行 變得非常重要。這是因為複雜算法的視覺化通常比數字輸出更容易理解。 資料視覺化如何運作 大多數據視覺化工具都具有連接到當前主流資料來源的連接機制，包括最常見的 關聯式資料庫、資料檔案、Hadoop 和各種雲端API 平台。視覺化工具從這些資料來 源中提取數據，透過適當的資料建模作業，進行資料的轉換後以圖形呈現 (有些工具 更能自動解釋資料相關性，然後將這些發現以最佳圖表類型呈現) 。另網際網路與手 持裝置的普及，現今的視覺化工具都會提供儀表板(Dashboard)組件，讓使用者將分 析結果透過Web/App 的方式呈現。 微軟視覺化工具:Power BI Microsoft Power BI 是微軟推出的一套商務分析工具套件，可提供使用者發展自 助式商業智慧，開發報表和儀表板，而無需依賴資訊人員或資料庫管理員。在本機環 境的作業支援方面，主要是以Power BI Desktop 版為主力，而在雲端環境的支援則 以Power BI Service 為主。其主要功能呈現如圖2.5-1 圖2.6-1 Power BI 功能架構資料來源:本研究整理 本研究以Power BI 做為視覺化分析工具，其過程包含從資料來源取得資料與清理 (使用PowerQuery)、資料建模(PowerPivot)到資料視覺化(使用PowerView)。如果報 表需要被分享，則可以發佈到Power BI Service 或Report Server 也就是微軟的雲 端及報表平台上。若不需要分享則使用Power BI Desktop 查看報表即可。 由於Power BI Service 只能讀取資料進行資料視覺化(PowerView)的功能，不能做 資料清理與轉換(PowerQuery)，也不能建立資料表之間的關聯、建立欄位與度量值， 寫DAX 公式等…(PowerPivot)，所以大部份情況會使用Power BI Desktop 將做好的 報表上傳到Power BI Service。至於在行動裝置的支援方面，目前 Android、IOS、 Windows 系統都針對Power BI App，可以透過行動裝置觀看來自於Power BI Service 的報表。 選擇Power BI 的理由 理由 說明 成本 免費版本的Power BI Desktop 可連接達90 幾種資料來源，使 用150 種圖表，功能全部免費。 功能 從資料連接、資料清理、資料建模到視覺化與報表分享（網頁、 手機、平板），功能齊全。 入門難度與產品 支援性 Power BI 的使用基礎概念源自於Excel，幾乎可以在不寫程式 的情況下上手使用。產品更新速度快，能持續因應市場並滿足 使用情境需求。 表2.6-1 Power BI 的特性 雖然Power BI 的使用基礎概念源自Excel，但它所使用的資料處理技術與Excel 不同。Power BI 的資料模型使用記憶體內分析引擎將資料儲存在記憶體中，執行強 大的壓縮技術以降低存取速度。引此平均來說資料大小會比Excel 小7 到10 倍左右。 第三章 研究方法 本研究以政府公開資料集及其年報資料作為本研究所需資料集，進行處理並產生人 口虛擬資料，此章節將對資料來源與處理方式以及使用之大數據技術進行資料，分析 與視覺呈現進行描述。整體研究架構如下圖3.1-1 所示。 圖3.1-1 本研究架構
